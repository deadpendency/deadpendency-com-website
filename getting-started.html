Deadpendency Getting Started

Installation

Go to the Deadpendency app on GitHub marketplace and install it. Select which repositories to install the GitHub app on.

How Deadpendency Works

  1. Push a commit to your repository. This could be a pull request (PR).
  2. The top 2 programming languages used in your repostiory are identified.
  3. The default dependency files of these programming languages are read.
  4. The dependencies are assessed.
  5. A report is produced. The 'check' will fail if the dependencies are not healthy.

Using Deadpendency

Start by pushing a commit to your repository. It may be best to start with a PR. Deadpendency will run, producing a report.

There are a few possible results from the report.

No Dependencies Found

This is due to either using a non-supported language, or more likely, your dependency files are in non-standard locations.

Refer to language support for the default dependency files loaded. You may need to configure additional dependency files in the Deadpendency config.

The Check Passes

This indicates that all your dependencies are actively being maintained. Deadpendency will continue to monitor your dependencies as new commits are pushed. Nothing to do here.

The Check Passes with Warnings

This indicates that some of your dependencies have health concerns. For example, there may be no commits within the last year.

Warnings serve to provide a quick report on dependencies that may become problematic in the future. It is recommended you configure (LINK ME) your failure thresholds so that only when the check fails does it a mean a dependency is unhealthy enough that it should be reviewed.

The Check Fails

This indicates that some of your dependencies have serious health concerns. For example, they may have been flagged as deprecated.

At this point you should review the failures. You then have a few options.

Ignore the failing dependencies

Sometimes there is no choice but to depend on an inactive or unhealthy dependency. For example, there may be no viable alternatives.

In this case it is recommended to ignore the dependency (LINK ME) from being checked by deadpendency.

Adjust your failure thresholds

Depending on your ecosystem, different failure thresholds for Deadpendency may make sense. Or you may decide to disable (or set as a warning) some checks entirely.

It is recommended that you tune the checks so that when failures happen, the dependency is definitely unhealthy enough to consider switching away.

Switch to an alternative

If a dependency is not actively maintained, then ideally you want to switch to a dependency which is.

Using Deadpendency Long Term

It is recommended to keep your Deadpendency passing, by ignoring or switching away from 'dead' dependencies.

Then, as you work on your project and push commits, Deadpendency will monitor your dependency health. You can largely ignore the report until you see a new failure.
