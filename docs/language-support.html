Deadpendency Language Support

Dependencies are currently only analyzed by Deadpendency when hosted on GitHub.

Deadpendency will only load the dependency files of the top 2 programming languages used in a repository. Additional dependency files can be configured.

Deadpendency will only search 1 directory deep for dependency files.

Programming Language Supported File Types Default Files Loaded Notes
JavaScript ✅ Npm package.json ./**/package.json
TypeScript ✅ Npm package.json ./**/package.json
Python ✅
  • Pip Requirements
  • Pipenv Pipfile
  • Pyproject.toml
  • ./**/requirements.txt
  • ./**/Pipfile
  • ./**/pyproject.toml
 Will not follow requirements.txt includes eg. -r other-requirements.txt.
pyproject.toml support includes basic and Poetry dependencies.
Ruby ✅
  • Bundler Gemfile
  • Ruby Gemspec
  • ./**/Gemfile
  • ./**/*.gemspec
 *.gemspec search only goes 1 level deep. Git blocks not yet supported.
PHP ✅ Composer file ./**/composer.json VCS dependencies not yet supported.
Rust ✅ Cargo file ./**/Cargo.toml Some dependency types not detected.
Haskell ✅
  • Cabal file
  • Hpack package.yaml
  • ./**/*.cabal
  • ./**/package.yaml
 There are some current limitations.
.NET ✅
  • PackageReference in project files
  • ./**/*.csproj
  • ./**/*.vbproj
 packages.config format is not supported.
Java ✅
  • Maven pom.xml
  • Gradle Build File
  • ./**/pom.xml
  • ./**/build.gradle
 Maven packages often don't reference the source repository. Only searches 1 directory deep for dependency files.
Kotlin ✅
  • Maven pom.xml
  • Gradle Build File
  • Gradle Kotlin DSL File
  • ./**/pom.xml
  • ./**/build.gradle
  • ./**/build.gradle.kts
 Maven packages often don't reference the source repository. Only searches 1 directory deep for dependency files.
Go ❌ Currently no clear registry API to determine source repository.
Others ❔ If you want a language supported please open an issue.
