Deadpendency Configuration

Report Truncation

The Deadpendency report is GitHub check run. The bulk of the report lives in the check run body, which has a (currently undocumented) limit of 65335 characters.

When a repository has enough dependencies in some cases it will exceed this limit. In this scenario Deadpendency will remove non-key information from the report. Currently, this only means that dependencies which are passing, will no longer inlcude links to the package registry or source repository.

Longer term, the Deadpendency report will be available external to GitHub to avoid the problem entirely.
