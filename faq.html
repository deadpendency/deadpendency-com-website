Deadpendency FAQ

What does Deadpendency detect?

It detects dependencies that have project health issues. For example is the dependency repository still actively being committed to.

If a dependency is marked as unhealthy, should I stop using it?

No! This is a call to action to review the dependency and make a decision as to whether to switch away or not.

In many cases it makes sense to keep using the dependency (eg. there are no viable alternatives). In this case, the dependency can be ignored from deadpendency checks.

What does BETA mean?

Deadpendency is newly developed. There are a lot of possible edge cases with many languages and registries. It will take some time and work handle these and the system to become very reliable.

Why isn’t Java supported?

Maven does not provide the source repository of its packages.

Why isn’t Other language supported?

Support is either not possible, planned, or the language is considered too niche. Feel free to create / up vote the relevant issue.

Which package registries support structured deprecation?

Currently NPM (JavaScript), Nuget (.NET) and Hackage (Haskell). .NET support is under development.

Why isn't Deadpendency open source?

The plan is to charge for Deadpendency for closed source. However, as much Deadpendency code as possible with be open sourced.

Why isn’t Deadpendency available for private repositories?

Deadpendency is in beta. The focus is on improving and stabilizing the system before making it available for purchase on private repositories.

How much will Deadpendency cost for private repositories?

To be determined. It should be cheap though.
