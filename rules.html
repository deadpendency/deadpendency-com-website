Deadpendency Rules

Deadpendency performs the following checks. See rules usage for details on using the rules config.

Rules that perform checks against the source repository currently only support GitHub repositories. More git hosts are planned to be supported.

Rule Description Warn at Fail at Config Notes
No Recent Commit No commits within X months. 3 months 6 months 
no-recent-commit:
  warn-at-months: 3
  fail-at-months: 6
 Cannot be configured to more than 24 months.
Few Yearly Commits Few total commits for the past year. 4-6 commits 1-3 commits 
few-yearly-commits:
  warn-at-count: 6
  fail-at-count: 3
No Recent Package Release No package release within X months. 6 months 9 months 
no-recent-package-release:
  warn-at-months: 6
  fail-at-months: 9
Repository Archived The source repository has been archived. N/A ✅ N/A Produces failure
Repository Is Fork The source repository is a fork. ✅ N/A N/A Produces warning
Package Deprecated The package is deprecated in the registry. N/A ✅ N/A Only Npm and Hackage.

More checks coming soon!
