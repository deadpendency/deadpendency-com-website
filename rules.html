Deadpendency Rules

Deadpendency performs the following checks.

The default settings are designed to be quite permissive and non-noisy. For more active ecosystems, you may wish to adjust the defaults. See rules usage for details on using the rules config.

Rules that perform checks against the source repository currently only support GitHub repositories. More git hosts are planned to be supported.

Rule Description Warn at Fail at Config Notes
No Recent Commit No commits within X months. 9 months 12 months 
no-recent-commit:
  warn-at-months: 9
  fail-at-months: 12
 Cannot be configured to more than 24 months.
Few Yearly Commits Few total commits for the past year. 2-3 commits 1 commit 
few-yearly-commits:
  warn-at-count: 3
  fail-at-count: 1
No Recent Package Release No package release within X months. 12 months 18 months 
no-recent-package-release:
  warn-at-months: 12
  fail-at-months: 18
Repository Archived The source repository has been archived. N/A ✅ N/A Produces failure.
Repository Is Fork The source repository is a fork. ✅ N/A N/A Produces warning.
Package Deprecated The package is deprecated (or flagged as abandoned) in the registry. N/A ✅ N/A Only Npm, Hackage, NuGet and Packagist.
Single Recent Author All the commits in the last year were authored by a single person. ✅ N/A N/A Produces warning.

More checks coming soon!
