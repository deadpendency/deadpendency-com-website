Deadpendency Usage

Example

See a detailed example of using Deadpendency.

Installation

Go to the Deadpendency app on GitHub marketplace and install it. Select which repositories to install the GitHub app on.

Usage

Once installed in a repository, the Deadpendency check will run on (the newest commit of) every git push.

Deadpendency will automatically search for dependency files based on the top 2 languages in use on your GitHub repository. It will then check the dependencies for project health and produce the Deadpendency report (see example). Unhealthy dependencies will trigger a check failure.

Refer to language support for information on what dependency files are supported and automatically loaded.

Configuration

Configuration is done by including a .deadpendency/config.yaml file in your GitHub repository source. This file is optional.

Additional Dependency Files

In some cases you have additional, or different dependency files to load than the default ones. These can be loaded like: 

additional-dependency-files:
  - type: pip-requirements-txt
    path: other-dependencies.txt

  - type: pipenv-pipfile
    path: custom/CustomPipfile
File Types

  • JavaScript Npm Package File - npm-package-json
  • Python Pip Requirements File - pip-requirements-txt
  • Python Pipenv Pipfile - pipenv-pipfile
  • PHP Packagist Compose File - packagist-composer-json
  • Ruby Bundler Gemfile - bundler-gemfile
  • Ruby Gemspec - rubygems-gemspec
  • Haskell Cabal File - haskell-cabal
  • Haskell Hpack File - hpack-package-yaml
  • Rust Crates Cargo File - crates-cargo-toml
  • C# .NET Project File - dotnet-csharp-project
  • Visual Basic .NET Project File - dotnet-visualbasic-project
  • Maven Pom Xml File - maven-pom-xml
  • Build Gradle File - build-gradle

Additional Dependencies

Useful when there are specific additional dependencies you want to be checked that are missing from your dependency files. Or for some reason Deadpendency is not detecting them.

additional-deps:
  javascript:
    # additional deps found in the package registry
    - react
    - react-dom

    # direct repository dependencies
    - repo: jquery/jquery
    # name can be included so Deadpendency can load the package details in the registry
    - name: random-on-github
      repo: some-org/super-random

  # unsupported languages can still have additional repository dependencies
  cool-lang:
  - name: cool-dep
    repo: cool/dep

Ignore Dependencies

Useful when you choose to depend on an unhealthy dependency.

ignore-failures:
  - react-dom
  - angular

Configure Deadpendency checks

Each language ecosystem is different. What might be considered unhealthy for a JavaScript dependency may be fine in a more niche language.

The default config
rules-config:
  no-recent-package-release:
    warn-at-months: 18
    fail-at-months: 24
  no-recent-commit:
    warn-at-months: 12
    fail-at-months: 18
  few-yearly-commits:
    warn-at-count: 2
    fail-at-count: false # does not fail by default

  # these are simply true / false for enable / disable
  repository-archived: true
  repository-is-fork: true
  package-deprecated: true
  single-recent-author: true
Customise the config

Unless a rules-config is provided, the defaults above are used. Settings can be overriden as per the below.

rules-config:
  no-recent-package-release:
    warn-at-months: 6
    # fail-at-months: 24  - omitted to use the default
  no-recent-commit:
    warn-at-months: 3
    fail-at-months: false # setting to false disables the failure check

  few-yearly-commits: false # or an entire check can be disabled

  single-recent-author: false # disable this check too

Note: the defaults may be adjusted from time to time.

See rules for detailed information on rules and defaults.
